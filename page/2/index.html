LeetCode783 二叉搜索树节点最小距离

题目：

　　给你一个二叉搜索树的根节点 root ，返回树中任意两不同节点值之间的最小差值。

提示：

　　树中节点数目在范围 [2, 100] 内

　　0 <= Node.val <= 105

题解

二叉搜索树（Binary Search Tree）：是一颗排序树，具有以下特点：

  1. 若它的左子树不空，则左子树上所有结点的值均小于它的根结点的值；
  2. 若它的右子树不空，则右子树上所有结点的值均大于它的根结点的值；
    它的左、右子树也分别为二叉排序树。

根据上述二叉搜索树的特点，可以得出结论：二叉搜索树的中序遍历是一种有序结构

  1. 先中序遍历存储在一个数组/list中，然后再遍历该容器取最小值
  2. 优化后的版本，减少了内存消耗以及一次遍历。这是因为二叉搜索树的有序特性决定了最小的距离应该在相邻节点，所以可以边遍历边计算。

什么是ziplist？

ziplist是一种特殊编码的双链表，其设计目的是提高内存效率。它同时存储字符串和整数值，其中，整数被编码为实际整数，而不是一系列字符。它允许在O(1)时间内在列表的任意一侧进行push和pop操作。但是，因为每个操作都需要重新分配ziplist所使用的内存，所以实际的复杂性与ziplist所使用的内存数量有关。
本质：字节数组

数据结构

  1. zlbytes：32位，表示整个ziplits所占的字节数，包含zlbytes本身占用的四个字节。
  2. zltail：32位，是到ziplist最后一个entry的偏移量，用于从尾部出队时的操作，避免整个链表遍历
  3. zllen：16位，表示ziplist中所有entry的数目。如果超过2^16-2，则设值为 2^16-1，然后需要通过遍历来获取ziplist中所有的entry数目。
  4. zlend：8位，是一个特殊的entry，设值为255（0xff）。其他项都不会以255编码开头。
  5. Entry：压缩节点
