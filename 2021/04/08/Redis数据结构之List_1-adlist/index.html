Redis数据结构之List_1-adlist

发表于 2021-04-08 分类于 Redis 热度： ℃ 热度： Valine：
本文字数： 5.4k 阅读时长 ≈ 5 分钟

前言

Redis的底层包含9种基本数据结构：

  1. String：
  2. List: 列表对象的编码可以是ziplist或者linkedlist。
  3. Hash
  4. Set
  5. Sorted Set
  6. bitmap
  7. GeoHash
  8. HyperLogLog
  9. Streams

本文侧重从源码的角度分析下List的底层数据结构之一——adlist.c。

什么是adlist？

adlist是 redis 自定义的一种双向链表，3.0及以前运用于list结构中，在3.2以后被quicklist取代。

源码分析

创建一个新的双端链表 listCreate

移除链表所有元素 listEmpty

该函数移除链表所有元素，但是并不释放链表结构体本身

相应的listRelease函数负责释放整个链表内存

新增链表节点

  1. 在链表头部添加节点： list *listAddNodeHead(list *list, void *value)
  2. 在链表尾部添加节点： list *listAddNodeTail(list *list, void *value)
  3. 根据after参数决定在old_node的前面还是后面插入。 after > 0： 在后面插入；当after = 0时，插入在前面

    删除链表节点

    需要注意的是待删除的节点是链表的头节点或者尾节点的特殊处理，最后别忘了释放该节点对应的内存空间

    迭代器

  4. 为链表初始化创建一个迭代器–listGetIterator
  5. 释放迭代器–listReleaseIterator
  6. 利用迭代器获取下一个元素–listNext

    链表复制

    listDup复制成功返回复制链表，如果出现OOM则返回NULL。无论复制成功与否，对原链表无影响。

    根据给定值查找链表节点

    listSearchKey函数从头节点开始查找链表节点，查到返回该节点否则返回NULL

    查找给定index的节点

    index >= 0时，正向查找，即index为0表示head节点；

    index < 0时，逆向查找，即index为-1表示tail节点。

    其他的一些函数

  7. 链表尾节点切割添加到链表头部： void listRotateTailToHead(list *list)
  8. 链表头节点切割添加到链表尾部： void listRotateHeadToTail(list *list)
  9. 链表拼接：void listJoin(list *l, list *o)。该函数会使链表o成为一个空链表。
